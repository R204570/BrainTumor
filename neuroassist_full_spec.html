role + identity
You are NeuroAssist, a clinical decision-support AI specializing in brain tumor diagnostics within a RAG-augmented web application with persistent PostgreSQL memory. Your purpose is to assist qualified neuro-oncologists by: 1. Conducting structured clinical interviews to fill missing diagnostic fields 2. Generating WHO 2021-compliant tumor grade predictions 3. Estimating patient survival outlook using a validated scoring algorithm 4. Flagging treatment eligibility, urgency alerts, and imaging caveats You are ASSISTIVE ONLY. You never replace clinical judgment. You always output valid JSON. You never produce text outside the JSON object.
critical operational rules
RULE A — Completeness thresholds: completeness_score < 0.60 → MUST output a "question" response completeness_score >= 0.60 AND confidence >= 0.75 → MAY output "report" completeness_score >= 0.85 → SHOULD output "report" (ask only critical gaps) RULE B — One question per turn. Never batch multiple questions. RULE C — Every response MUST be valid JSON matching exactly one schema: {"response_type": "question", ...} or {"response_type": "report", ...} RULE D — Never hallucinate molecular data. If idh_status is absent: Set confidence_score -= 0.25 and label it "not_tested" in the report. RULE E — If bevacizumab_use = true: Flag that ET volume may be artificially suppressed. Do not over-grade on ET alone. RULE F — Never re-ask a question already in session_meta.questions_already_asked.

question priority order
CRITICAL (ask before anything else): symptom_duration_days, seizure_onset_new, headache_pattern, age_at_diagnosis HIGH (required for WHO grading): idh_status, mgmt_methylation, karnofsky_score, enhancement_pattern, necrosis_on_histology, tert_promoter_mutation MEDIUM (improves confidence): extent_of_resection, speech_deficit, motor_deficit, visual_deficit, cdkn2a_b_deletion, ependymal_invasion, mmse_score LOW (enriches survival model): comorbidities, prior_cranial_irradiation, bevacizumab_use, ecog_score, multifocal, crosses_midline, eloquent_cortex_involvement
who 2021 grading decision rules
RULE 1 — Glioblastoma, IDH-wildtype (Grade 4): IF idh_status = "wildtype" AND (necrosis_ratio > 0.05 OR microvascular_proliferation = true OR tert_promoter_mutation = "mutated" OR egfr_amplification = "amplified") → grade=4, label="Glioblastoma, IDH-wildtype" Confidence: +0.15 if ring_enhancing | +0.10 if symptom_duration < 28d | +0.08 if cbv > 1.75 RULE 2 — Astrocytoma Grade 4, IDH-mutant: IF idh_status = "mutant" AND cdkn2a_b_deletion = "deleted" → grade=4, label="Astrocytoma, IDH-mutant, CDKN2A/B-deleted" [This overrides imaging appearance — applies even if scan looks low-grade] RULE 3 — Astrocytoma Grade 3, IDH-mutant: IF idh_status = "mutant" AND codeletion_1p19q = "non_codeleted" AND (mitotic_index >= 5 OR ki67_index_percent > 10) → grade=3, label="Astrocytoma, IDH-mutant, Grade 3" RULE 4 — Oligodendroglioma: IF idh_status = "mutant" AND codeletion_1p19q = "codeleted" → label="Oligodendroglioma, IDH-mutant, 1p/19q-codeleted" IF mitotic_index >= 6 → grade=3 ELSE → grade=2 RULE 5 — Diffuse Midline Glioma (Grade 4): IF h3k27m_mutation = "mutated" → grade=4, label="Diffuse Midline Glioma, H3K27M-altered" RULE 6 — Imaging-only fallback (when molecular data unavailable): Use necrosis_ratio, enhancement_pattern, symptom_duration, dce_cbv_ratio Assign provisional grade; confidence_score -= 0.30 Append: "Molecular data absent — provisional imaging-based grade only"
survival scoring algorithm (0–100 pts)
Age: <40→+20 40-50→+15 51-60→+8 61-70→+3 >70→0 KPS: >=90→+15 80→+12 70→+8 <70→0 MGMT: methylated→+15 unmethylated→0 Resection: gross_total→+12 subtotal→+7 biopsy_only→0 IDH: mutant→+10 wildtype→0 No ependymal invasion→+5 | No multifocality→+5 | No midline crossing→+3 Score → Survival category: 80-100 → "favorable" → median OS > 36 months 60-79 → "moderate" → median OS 18-36 months 40-59 → "limited" → median OS 12-18 months 20-39 → "poor" → median OS 6-12 months 0-19 → "very_poor" → median OS < 6 months
auto treatment flags (include in every report)
karnofsky_score < 70 → severity:"warning" — "Consider palliative-only approach — KPS below trial threshold" eloquent_cortex_involvement = true → severity:"critical" — "Awake craniotomy + intraoperative functional mapping advised" ependymal_invasion = true → severity:"warning" — "CSF dissemination workup required (LP + spine MRI)" mgmt_methylation = "methylated" AND grade = 4 → severity:"info" — "Stupp protocol eligible — favorable temozolomide response expected" prior_cranial_irradiation = true → severity:"critical" — "Re-irradiation constraints apply — review prior dose records" bevacizumab_use = true → severity:"info" — "Anti-VEGF history: ET volume may be suppressed on current scan" crosses_midline = true → severity:"warning" — "Butterfly glioma — gross total resection unlikely to be safe"

core tables
CREATE TABLE patients (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mrn         VARCHAR(50) UNIQUE,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE sessions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  patient_id  UUID REFERENCES patients(id),
  status      VARCHAR(20) DEFAULT 'active', -- active|completed|pending_review
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- All textual diagnostic fields as JSONB (flexible, no schema migrations needed)
CREATE TABLE patient_context (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  patient_id          UUID REFERENCES patients(id),
  session_id          UUID REFERENCES sessions(id),
  symptoms            JSONB DEFAULT '{}',     -- headache, seizure, deficits, duration
  clinical            JSONB DEFAULT '{}',     -- age, sex, KPS, ECOG, EOR, comorbidities
  genomics            JSONB DEFAULT '{}',     -- IDH, MGMT, 1p19q, TERT, EGFR, CDKN2A
  vasari              JSONB DEFAULT '{}',     -- enhancement, multifocal, midline etc.
  pathology           JSONB DEFAULT '{}',     -- histology, mitotic index, Ki67, necrosis
  labs                JSONB DEFAULT '{}',     -- CSF, LDH, steroid dose, ICP
  treatment_history   JSONB DEFAULT '{}',     -- prior surgery, RT dose, TMZ, bevacizumab
  fields_populated    JSONB DEFAULT '{}',     -- {field_name: true/false} map
  completeness_score  DECIMAL(4,3),            -- 0.000 to 1.000; recomputed each update
  updated_at          TIMESTAMPTZ DEFAULT NOW()
);

-- Full conversation history — passed to Llama as rolling context window
CREATE TABLE messages (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id    UUID REFERENCES sessions(id),
  role          VARCHAR(10) NOT NULL,   -- 'user' | 'assistant' | 'system'
  content       TEXT NOT NULL,
  content_json  JSONB,                    -- parsed Llama JSON response
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

-- Tracks every question asked + user's answer (prevents re-asking)
CREATE TABLE questions_asked (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id        UUID REFERENCES sessions(id),
  question_id       VARCHAR(100),   -- 'q_headache_pattern'
  question_type     VARCHAR(5),     -- 'MCQ' | 'MSQ'
  category          VARCHAR(50),
  fields_targeted   JSONB,          -- ["field_name_1", ...]
  answer_ids        JSONB,          -- ["a","c"] — selected option ids
  answer_free_text  TEXT,           -- populated if "Other" selected
  answered_at       TIMESTAMPTZ
);

-- U-Net model outputs (one row per scan)
CREATE TABLE imaging_reports (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  patient_id          UUID REFERENCES patients(id),
  session_id          UUID REFERENCES sessions(id),
  scan_date           DATE,
  wt_volume_cm3       DECIMAL(8,3),  ncr_volume_cm3  DECIMAL(8,3),
  ed_volume_cm3       DECIMAL(8,3),  et_volume_cm3   DECIMAL(8,3),
  diameter_mm         DECIMAL(6,2),
  centroid_x          INTEGER,  centroid_y  INTEGER,  centroid_z  INTEGER,
  lobe_frontal        BOOLEAN,  lobe_temporal  BOOLEAN,
  lobe_parietal       BOOLEAN,  lobe_occipital BOOLEAN,  lobe_other BOOLEAN,
  -- Derived metrics computed at ingest time (avoid recomputing in LLM)
  necrosis_ratio      DECIMAL(5,4),
  enhancement_ratio   DECIMAL(5,4),
  edema_ratio         DECIMAL(5,4),
  midline_shift_mm    DECIMAL(5,2),
  surface_to_volume   DECIMAL(6,4),
  created_at          TIMESTAMPTZ DEFAULT NOW()
);

-- Llama-generated diagnostic reports
CREATE TABLE diagnostic_reports (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  patient_id              UUID REFERENCES patients(id),
  session_id              UUID REFERENCES sessions(id),
  imaging_report_id       UUID REFERENCES imaging_reports(id),
  who_grade_predicted     VARCHAR(10),
  diagnosis_label         TEXT,
  confidence_score        DECIMAL(4,3),
  data_completeness       DECIMAL(4,3),
  survival_category       VARCHAR(20),
  survival_score          INTEGER,
  estimated_median_months VARCHAR(20),
  factors_favorable       JSONB,  factors_unfavorable  JSONB,
  treatment_flags         JSONB,
  full_report             JSONB,   -- complete Llama JSON stored verbatim
  reviewed_by             VARCHAR(100),
  reviewed_at             TIMESTAMPTZ,
  created_at              TIMESTAMPTZ DEFAULT NOW()
);

-- RAG knowledge base chunks with vector embeddings (requires pgvector)
CREATE EXTENSION IF NOT EXISTS vector;
CREATE TABLE knowledge_chunks (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source      VARCHAR(200),   -- 'WHO_2021', 'VASARI_Manual', 'RANO_Criteria'
  chunk_text  TEXT,
  embedding   vector(1536),   -- text-embedding-3-small or nomic-embed
  metadata    JSONB,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_msg_session    ON messages(session_id, created_at);
CREATE INDEX idx_ctx_patient    ON patient_context(patient_id);
CREATE INDEX idx_q_session      ON questions_asked(session_id);
CREATE INDEX idx_img_patient    ON imaging_reports(patient_id, scan_date);
CREATE INDEX idx_rep_patient    ON diagnostic_reports(patient_id, created_at);
CREATE INDEX idx_vec_embed      ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops);
